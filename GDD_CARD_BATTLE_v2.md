# 卡牌对战游戏 - 设计文档 v2.0

## 一、核心机制设计

### 1.1 新增卡牌属性

```
基础属性：
- attack      : 物理攻击（受护甲减伤）
- defense     : 物理防御（减少物理伤害）
- magic_attack: 魔法攻击（受魔法防御减伤）
- magic_defense: 魔法防御（减少魔法伤害）
- health      : 生命值
- energy      : 能量消耗

特殊属性（整数或百分比）：
- real_damage     : 真实伤害（无视一切防御）
- reflect_damage  : 反伤比例（受到攻击时反弹%伤害）
- shield          : 护盾值（额外生命，先扣护盾）
- heal            : 治疗量（出牌时回复）
- lifesteal       : 吸血比例（造成伤害时回复%生命）
- vulnerability    : 易伤比例（受到伤害+%，负数=减伤）
- armor           : 护甲值（减少物理伤害，独立的减伤机制）
```

### 1.2 伤害计算公式

```
# 物理伤害计算
物理伤害 = max(1, 攻击方物理攻击 - 受击方护甲 - 受击方防御) 

# 魔法伤害计算
魔法伤害 = max(1, 攻击方魔法攻击 - 受击方魔法防御)

# 真实伤害计算
真实伤害 = 攻击方真实伤害值（无视一切减伤）

# 最终伤害
if 有护盾:
    扣除护盾值
else:
    最终伤害 = (物理伤害 + 魔法伤害 + 真实伤害) × (1 + 易伤比例)
    if 反击触发:
        攻击方受到 反击伤害 = 最终伤害 × 反击比例
    if 吸血触发:
        攻击方回复 = 造成伤害 × 吸血比例
```

## 三、战斗系统设计

### 3.1 战斗流程优化

```
回合开始：
1. 抽1张牌（上限6张）
2. 能量回复2点（上限7点）
3. 刷新护盾（不清空，保留到被打破）
4. 清除上回合的负面状态

玩家行动阶段：
1. 选择并打出卡牌（可多张）
2. 可以撤回已出牌（返还能量）
3. 点击结束回合

AI行动阶段：
1. AI根据策略出牌
2. 展示AI出牌动画

战斗结算阶段：
1. 计算所有出牌的总效果
2. 同时应用伤害（先扣护盾）
3. 处理反伤（同时生效）
4. 处理吸血（同时生效）
5. 判断胜负

回合结束：
1. 检查生命值
2. 进入下一回合
```

### 3.2 UI显示优化

战斗界面新增显示：

## 四、数值平衡设计

### 4.1 能量曲线

```
玩家初始能量：3点
每回合回复：2点
能量上限：7点

卡牌能量分布：
- 1费卡：3张（早期过渡）
- 2费卡：8张（主力）
- 3费卡：6张（中期爆发）
- 4费卡：3张（终结手段）
```

### 4.2 生命值曲线

```
玩家生命：100点（固定）
关卡1敌人：80点
关卡2敌人：90点
关卡3敌人：100点
关卡4敌人：110点
关卡5敌人：130点
```

### 4.3 预期战斗时长

```
- 每回合出牌数：1-2张
- 平均每张卡伤害：10-15点
- 预期回合数：5-8回合
- 每局游戏时长：2-4分钟
```

## 五、技能触发时机

### 5.1 出牌时触发（立即）

```
- heal: 立即回复生命
- shield: 立即获得护盾
- 对目标的debuff: 立即生效
```

### 5.2 造成伤害时触发

```
- lifesteal: 伤害造成后回复
- 触发时机：物理伤害+魔法伤害+真实伤害结算后
```

### 5.3 受到伤害时触发

```
- reflect_damage: 受到攻击后立即反击
- 触发时机：在扣除护盾/生命后
```

## 六、设计支柱（Design Pillars）

```
1. 【策略深度】- 每张卡都有独特价值，需要玩家思考组合
2. 【职业特色】- 物理/魔法/功能三条路线，玩家可选择偏向
3. 【敌人特色】- 每关敌人机制不同，考验玩家适应能力
4. 【可读性】- 卡牌效果直观，新手友好
5. 【平衡性】- 没有无敌卡组，每关都需要不同策略
```
